A few tips

#Create a new Article

First of all, you must be connected to create a new article then you can either go to dashboard/article-list page or type directly into the url bar to create one.
A new article must belong to a category and have a proper url (Url must contains only letter, number, dash). If you choose to create a new article typing directly from the url bar, make sure the caregory name exists and the url isn't already use (ex: default/my-new-test-article). If an article already exists in the choosen category, C[E]MS will display it.

You cannot update an article if it does not belong to your account. Even Admin user cannot modify an article that he/she didn't create.

The article owner name will be highlighted in blue and a save button will appear at the end of it if the article belongs to your account

C[E]MS use content-editable attribute. A new article will by default be filled with lorem ipsum text. you just have select it and modify it.

You can add/remove content by double-clicking an editable text. A panel will be displayed. You can add a new div or h3, or delete the current selected node

You can save the modification on an article by clicking the save button at the end of it or use the [alt-s] shortcut

C[E]MS use canvas for image modification. A new article will by default contains a black image that you can modify. Just drag and drop a new image on it to open a bitmap editor

Remember: saving a new image from the editor will only update it. To link it to the current article, you have to save the article by clicking the save button or using the [alt-s] shortcut

#Validate a new Article

Once a new article has been created, only Admin user can activate it

This can be achieved by accessing dashboard/article-list as Admin

It won't be displayed for anyone other than Admin and article owner until it is activated

#Create a new User

When launching the application for the first time, the account created will automatically be Admin. This is the only Admin account possible

When you create another account via the application banner, this account will not be active until Admin validate it

You can achieve it by going to the dashboard/user-list and activate the desired account

A deactivated account can't create new content or access its dashboard.

#Create/delete a Category

When launching the application for the first time, 3 categories will be automatically created (default, cat1 and cat2)

You can add/delete a category except the default one. Articles that belong to a recently deleted category will be automatically assigned to the default one

Only Admin can create new category

#Shortcut list

  • [alt-l] : display/hide the login module
  • [alt-h] : go to homepage
  • [alt-p] : go to dashboard/user-info (connected only, 401 otherwise)
  • [alt-q] : disconnect
  • [alt-s] : saving the article modification

#401

If you hit a 401 page you will be automatically logged out. But this will happen if you try something not allowed...

#Email

C[E]MS does't provide a smtp server but bcrypt the passwords before saving them. So there's no way for us to give you the possibility to access your dashboard if you forget tour password.
C[E]MS will soon put a nodemailer service to correct this problem.

#Reset

You can reset the application by typing in command line gulp-reset. This will drop the database and destroy all uploaded images. Mongod must be running to reset the application.
